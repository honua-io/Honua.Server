**Guidelines for this template:**

1. **Be Specific**: Avoid vague statements. Use concrete examples and data.
2. **Be Honest**: Document both pros and cons objectively.
3. **Be Complete**: Consider at least 2-3 alternatives before deciding.
4. **Be Actionable**: Readers should understand what to do after reading.
5. **Link to Code**: Connect decisions to actual implementation.
6. **Keep Updated**: If status changes, update the ADR (don't delete history).

**Before submitting:**
- [ ] Title clearly describes the decision

- [ ] Status is appropriate (usually "Proposed" for new ADRs)
- [ ] Context explains why this decision is needed
- [ ] Decision is clear and actionable

- [ ] Consequences are honest and complete
- [ ] Code references are accurate
