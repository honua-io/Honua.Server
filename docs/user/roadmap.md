# Roadmap & Voting

Provide feedback or vote on features via GitHub Discussions: <https://github.com/honua/honua.next/discussions/categories/roadmap>

## Phase 0 (MVP)
- ✅ OGC API Features core plus transactional create/update/delete with GeoJSON, KML, GeoPackage, and tiles.
- ✅ Geoservices REST a.k.a. Geoservices REST a.k.a. Esri REST FeatureServer compatibility, including applyEdits pipelines.
- ✅ SQLite, PostGIS, SQL Server providers with read/write CRUD support.
- ✅ Styling MVP + catalog refresh.
- ✅ Support CLI, telemetry opt-in, compliance docs.

## Phase 1 Candidates
- [ ] CLA automation & public contributor onboarding.
- [ ] Advanced styling editor + legend services.
- [ ] Bulk import/ETL workflows (GeoPackage → database).
- [ ] Real-time event streaming / geofencing.
- [ ] Mapbox/Kepler integrations with dynamic expressions.

## How to Propose a Feature
1. Create a discussion in the Roadmap category with background, use case, and proposed scope.
2. Share relevant telemetry/metrics or support bundle IDs if applicable.
3. Encourage teammates to vote (👍).
4. Product team triages discussions during roadmap reviews and updates this document.

For bugs or support requests, open a GitHub issue using the appropriate template.
