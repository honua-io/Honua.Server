 10/// Unlike OFFSET pagination which scans all previous rows, keyset pagination uses a WHERE clause based on
 11/// the last seen value, resulting in consistent O(1) performance for all pages.
 12/// </summary>

 14/// Performance characteristics:
 15/// - OFFSET pagination: Page N takes O(N) time due to scanning N*pageSize rows

 17///

 19/// - OFFSET page 1: ~10ms

 21/// - OFFSET page 1000: ~10000ms (1000x slower)
 22/// - Keyset page 1: ~10ms
